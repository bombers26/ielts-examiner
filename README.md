# ielts-examiner: 学生自上传资料功能

## 概述

你可以将自己收集的雅思备考资料上传给本 skill，它将帮你整理、提取可复用规则、与官方标准对齐，并生成个人化训练建议。上传的资料不会被写入原始资料文件，仅在此次会话中使用。

---


> 使用注意：不同宿主的 skill discovery 机制不同。请在已启用 skill discovery 的普通交互会话中测试本 skill，不要使用会跳过 skill 加载的 bare / no-skill 模式。
## 如何上传

在对话中直接发送以下句式即可触发资料导入模式：

| 意图 | 示例 |
|------|------|
| 上传资料 | "我上传了雅思资料" + 粘贴文本/拖入文件 |
| 机构讲义 | "根据这份机构讲义训练我" + 粘贴内容 |
| 个人笔记 | "帮我整理这份雅思笔记" + 粘贴内容 |
| 错题/作文/口语稿 | "根据我的错题做诊断" / "根据这篇口语稿给反馈" + 粘贴内容 |

## 本地资料目录

额外资料和用户练习原稿统一放在 `materials/`，不再单独拆出“个人备考资料”一类。

```text
materials/
├── listening/
│   ├── extra_materials/
│   └── user_practice_drafts/
├── speaking/
│   ├── extra_materials/
│   └── user_practice_drafts/
├── reading/
│   ├── extra_materials/
│   └── user_practice_drafts/
└── writing/
    ├── extra_materials/
    └── user_practice_drafts/
```

- `extra_materials/`：官方样题、评分标准、taxonomy、第三方讲义、样例反馈。
- `user_practice_drafts/`：用户自己的作文原稿、二稿、口语转写、听力/阅读错题表。

## 支持格式

| 格式 | 说明 | 示例 |
|------|------|------|
| **纯文本** | 直接粘贴到对话中 | 作文正文、口语文字稿、笔记片段 |
| **Markdown** | 结构化笔记、讲义 | 培训机构 PDF 转制为 .md |
| **附件** | 通过当前宿主支持的附件能力上传 | .pdf, .docx, .txt, .md |

> 注：附件上传能力取决于当前宿主支持。若附件不可读，请把关键内容转成文本或 Markdown 粘贴。

## 资料可信度分级

上传后，本 skill 会对资料进行来源可信度判断：

| 等级 | 标签 | 说明 | 冲突处理 |
|------|------|------|----------|
| **T1** | 官方来源 | BC / IDP / Cambridge 官方发布 | 以此为准 |
| **T2** | 半官方 | 剑桥真题集、British Council 授权教材 | 可参考，与官方标准冲突时以官方为准 |
| **T3** | 高质量第三方 | 知名培训机构讲义、高分考生笔记 | 提取可用部分，标注可能偏差 |
| **T4** | 用户自制 | 个人笔记、错题集、自编练习 | 作为训练线索，不与官方标准直接比较 |
| **T0** | 无法判断 | 来源不明或无法确认 | 标注"来源不可验证，请谨慎使用" |

## 隐私边界

- **不上传官方文件**：你不应上传受版权保护的 IELTS 官方真题 PDF。本 skill 不存储、不转发你的文件
- **临时处理**：上传的资料在当前会话中使用，不持久化到 skill 目录或外部存储
- **建议脱敏**：如资料包含个人信息（姓名、考试日期、考号），建议在上传前脱敏
- **无数据收集**：本 skill 不收集、不上传你的资料内容。所有处理在本地会话中完成

## 示例用法

### 示例 1：上传机构写作模板

> 你：根据这份机构讲义训练我。它说 Task 2 要写五段，第一段背景+观点+预告。
>
> 助手：收到讲义。你的材料主张五段结构（T3 — 第三方机构）。对照官方评分标准（T1）：
> - 官方要求：Task Response 看重观点展开，不强制段落数
> - 建议：五段可用，但要保证每个正文段有充分展开
> - 冲突注意：你讲义里的"万能模板句"（It is crystal clear that...）在官方标准中被视为机械套用

### 示例 2：上传口语错误笔记

> 你：帮我整理这份雅思口语笔记。我记了一些 Part 2 的结构套路。
>
> 助手：收到笔记。提取到以下可复用规则：
> 1. Part 2 按"what → when → where → why + feeling"组织（与官方 FC 标准一致）
> 2. 推荐过渡句 "I'd like to talk about..."（自然，不扣分）
> ⚠️ 注意：你的笔记提到"必须用三个高级词汇"—此类策略可能导致 LR 分数异常波动，建议以自然准确优先

### 示例 3：根据错题出训练计划

> 你：根据我的错题做诊断。我在 Reading 中 TF/NG 题总是错。
>
> 助手：收到你的错题记录。诊断：
> - 误判模式：你把 NG 当成了 False（很常见，占你错题的 70%）
> - 训练建议：Day 1-3 专攻 TF/NG 题型，用 Cambridge 真题练习

## 冲突处理原则

当你的资料与本 skill 内置的官方标准冲突时：

1. **官方标准（T1）优先**：所有评分和建议以 IELTS 官方 band descriptors 为准
2. **差异标注**：会明确指出冲突点，说明为什么以官方为准
3. **保留可用部分**：不因为部分冲突就完全抛弃你的资料

## 雅思教学老师模式

除了评分，本 skill 也可以作为“雅思教学老师”使用。适合以下情况：

| 需求 | 示例 |
|------|------|
| 持续训练 | “按我的弱点带我练 7 天” |
| 二稿复盘 | “这是我改完的版本，只看上一轮问题有没有修好” |
| 资料内化 | “根据这份机构讲义，提炼我能用的训练动作” |
| 错题归因 | “这些 Reading 错题帮我归因并安排训练” |

教学老师模式的固定输出：

1. 当前最大瓶颈
2. 证据
3. 示范修改或处理方法
4. 一个最小训练任务
5. 验收标准

它不会把所有问题一次性倒给你；优先解决最限制分数的 1-2 个问题。
## 不支持

- ❌ 上传整个真题 PDF（受版权保护，且本 skill 不存储文件）
- ❌ 要求"按这本教材逐章讲课"（本 skill 是反馈助手，不是网课系统）
- ❌ 要求批量比对多个第三方材料的优劣（超出单一 skill 能力范围）
