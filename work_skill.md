## 一、Writing Task 1 评估细则

### 1. Task Achievement

**检查清单：**
- [ ] 是否改写了题目（paraphrase）
- [ ] 是否有 overview（回答"整体趋势是什么、最高/最低是什么"）
- [ ] overview 是否只写 1-2 句、不含具体数据
- [ ] 是否选出了 3-4 个主要特征
- [ ] 关键数据是否准确支持判断
- [ ] 是否避免了逐年流水账

**Band 锚点：**
- Band 5: 没有 overview，逐项报数，数据无关或无比较
- Band 6: 有 overview 但不够清晰，主要特征选出了部分，比较不足
- Band 7: overview 清晰，选材合理，数据有效支持主要特征
- Band 8: 高度概括，信息选择精准，overview 覆盖整体趋势和例外

### 2. Coherence and Cohesion

**检查清单：**
- [ ] 是否有明确的四段结构（Intro / Overview / Body 1 / Body 2）
- [ ] 段落内部逻辑是否清晰（从总到分）
- [ ] 衔接手段是否自然（By contrast, Meanwhile, Similarly）
- [ ] 是否过度使用衔接词（每个句子都加 Firstly / Secondly）
- [ ] 代词 and 替换是否清晰（无歧义）

### 3. Lexical Resource

**检查清单：**
- [ ] 趋势词是否准确（rise, increase, decline, fluctuate）
- [ ] 程度副词是否匹配实际变化（sharply vs slightly）
- [ ] 比较结构是否正确（twice as high as, overtook, the gap widened）
- [ ] 比例表达是否准确（accounted for, made up, comprised）
- [ ] 是否有概念混淆（percentage 和 number 混用）
- [ ] 是否有中式表达或成语直译

**推荐词汇体系（基于本地资料库）：**
- 上升：rise, increase, grow, climb, surge, soar, jump
- 下降：fall, decline, drop, decrease, dip, plunge
- 稳定：remain stable, stay constant, level off, plateau
- 波动：fluctuate, vary
- 比较：A was twice as high as B, A overtook B, A remained ahead of B, the gap widened considerably
- 比例：accounted for, represented, constituted, made up
- 地图：be replaced by, be converted into, be expanded, be relocated, be demolished
- 流程：be produced, be heated, be mixed with, pass through, be converted into

### 4. Grammatical Range and Accuracy

**检查清单：**
- [ ] 时态是否与时间轴一致（过去图用 past simple）
- [ ] 主谓一致是否稳定
- [ ] 比较结构是否正确
- [ ] 是否有分词结构（rising steadily, followed by）
- [ ] 被动语态是否正确（流程题/地图题）
- [ ] 标点是否正确（句号、逗号、分号使用规范）

### 题型专项（内置专项规则）

**Line graph：** 总趋势 + 交叉点（overtook / surpassed），按上升/下降分组
**Bar chart：** 最高/最低 + 分组对比，无时间轴时不写 trend
**Pie chart：** 最大份额 + 最小份额 + 合并比例
**Table：** 极值 + 行/列模式，不逐格写
**Mixed charts：** 共同 overview 覆盖两个图，不写成两篇
**Process diagram：** 被动语态 + 顺序连接词，不编数据
**Map：** 方位词 + 变化动词（added, removed, replaced, expanded）

## 二、Writing Task 2 评估细则

### 1. Task Response

**检查清单：**
- [ ] 是否讨论了题目要求的所有方面
- [ ] 立场是否清晰且一致
- [ ] 是否每个主体段都有明确的论点
- [ ] 论点是否有展开（举例 / 因果推理 / 对比）
- [ ] 结论是否与正文立场一致
- [ ] 是否偏题或只讨论了一半

**常见问题：**
- 只讨论优点或缺点的一边（discuss both views 不完整）
- 引言套话太长，迟迟不进入正题
- 结论引入新论点

### 2. Coherence and Cohesion

**检查清单：**
- [ ] 段落数是否合理（4-5 段）
- [ ] 每个段落是否有明确主题句
- [ ] 段落间是否有逻辑递进
- [ ] 衔接词是否自然（While I concede that... I firmly believe that...）
- [ ] 是否过度使用 First(ly), Second(ly), Third(ly)

### 3. Lexical Resource

**检查清单：**
- [ ] 是否准确使用学术词汇（efficiency, significantly, inevitable）
- [ ] 是否有低级拼写错误（academic → acdamic, efficiency → effience 等）
- [ ] 是否有中式直译（"历史车轮滚滚向前" → 删掉）
- [ ] 是否有生造词（double-blend sword → double-edged sword）
- [ ] 搭配是否地道（divert attention, not corrupt attention）

### 4. Grammatical Range and Accuracy

**绝症级错误（直接限制 6.0 以下）：**
- Comma splice：用逗号连接两个完整句子 → 改为句号 / 分号 / 连词
- 主谓不一致：technology play, problems is → 单数谓动加 s
- to 不定式后接动名词：drive companies to producing → drive companies to produce
- 词性混用：environment problem → environmental problem

## 三、Speaking 评估细则

### 仅文字稿时的限制

- Fluency and Coherence：可评估（留意自我纠正频率、犹豫标记、重复）
- Lexical Resource：可评估（词汇范围、paraphrase 能力、搭配准确度）
- Grammatical Range and Accuracy：可评估（句式复杂度、错误频率）
- Pronunciation：**无法可靠评分** — 必须标注"需要录音才能评估"

### 常见问题识别
- 过度使用填充词（well, you know, I mean, like）
- 话题展开不足（只给 yes/no，不延展）
- 死记硬背整段回答（语调模式缺失，但从文本难以判断）
- 回答与问题无关

## 五、训练建议

可推荐的训练路径：
- **每日流程：** 2min 读题 → 3min 中文提纲 → 20min 限时写作 → 5min 自查
- **14 天计划：** Day 1 评分标准 → Day 2-10 题型专项 → Day 11-13 限时整篇 → Day 14 复盘
- **自评分档参考：**
  - 5.5-6.0：基本描述但 overview 弱、比较少、错误影响理解
  - 6.5：主要特征基本清楚但发展不充分
  - 7.0：overview 清晰、选材合理、比较充分、错误不阻碍理解
  - 7.5+：组织自然、语言灵活准确、数据支撑克制有效

**错误记录表推荐：**
| 日期 | 题型 | 词数 | 用时 | overview 问题 | 数据问题 | 语法问题 | 下次重点 |

## 六、学生自上传资料处理细则（Student Material Intake）

### 6.1 资料类型识别

| 类型 | 识别特征 | 典型内容 |
|------|----------|----------|
| **机构讲义** | 结构化、标题层级、含"考点""技巧""保分"等词 | 培训机构 PDF / 课堂笔记 |
| **个人笔记** | 非正式格式、缩写、中英混杂、个人标注 | Notion / 手写转文字 |
| **真题回忆** | 题目描述片段、无完整原文 | 考后回忆贴 |
| **错题集** | 表格或列表、错答 + 正答对比 | Listening/Reading 错题汇总 |
| **写作练习** | 完整作文 + 可能含题目 | 自行写作或批改后版本 |
| **口语稿** | Part 1/2/3 标注、对话式文字 | 练习录音转写 |
| **备考计划** | 日程表、目标分数、时间分配 | 学习计划表 |
| **宣传材料** | 含"保分""提分 X 天""命中率"等营销用语 | 机构广告页 |

### 6.2 来源可信度分级

| 等级 | 标准 | 处理方式 |
|------|------|----------|
| **T1（官方）** | ielts.org / British Council / IDP / Cambridge 官网发布 | 可作核心依据 |
| **T2（半官方）** | Cambridge 真题集、BC 授权教材 | 可参考，标注来源 |
| **T3（第三方）** | 知名培训机构、高分考生分享 | 提取可用规则，标注潜在偏差 |
| **T4（自制）** | 用户个人笔记、错题集 | 作为训练线索，不引用为"标准" |
| **T0（未知）** | 无法识别来源 | 标注"来源不可验证，请谨慎使用" |

默认不带来源标注的上传内容归类为 T3-T4 之间，除非有明显官方来源特征。

### 6.3 规则提取与对齐流程

```
用户上传 → 类型识别 → 可信度分级 → 提取可复用规则
                                          ↓
                             逐条与官方标准对比
                                          ↓
              ┌────────────┬────────────┬────────────┐
              ✅ 一致      ⚠️ 部分偏差   ❌ 冲突
              ↓             ↓             ↓
          推荐保留    标注偏差原因    引用官方标准
                       建议修正      建议以官方为准
                                          ↓
                             输出个人化训练建议
```

### 6.4 冲突处理规则

1. **T1 > 一切**：任何来源与官方标准冲突时，必须以官方标准为准
2. **T2 + 官方交叉验证**：T2 材料的方法学建议可作为练习参考，但不作为评分标准
3. **T3 分场景评估**：T3 材料的"速成技巧"在低分段可能有效，在高分段可能成为扣分点
4. **T4 不推广**：用户自制的个人策略不保证对他人有效，不主动推荐给其他用户
5. **T0 标注风险**：来源不明标注使用风险，不建议作为核心备考依据

### 6.5 常见场景处理模板

**场景 1：用户上传的模板与官方标准冲突**
> 你的资料建议"每个句子都用 Firstly / Secondly / Thirdly"。官方 CC 评分标准认为过度使用序列连接词是不自然的衔接方式。建议改为多样化衔接：一些段落用 First of all 开头，另一些用 Meanwhile / By contrast / Turning to 自然过渡。

**场景 2：用户资料中的词汇建议不准确**
> 你的讲义把 percentage 和 number 互换使用。在官方评分标准中这两个概念不同：percentage 是百分比，number 是具体数值。Task 1 中混用会导致数据描述不准，影响 TA 分数。

**场景 3：用户笔记中有效的个人策略**
> 你笔记中"Part 2 按 when → where → what → why 组织"的策略与官方 FC 标准中"话题展开自然有序"的要求一致。建议保留。

### 6.6 隐私与数据处理

- 用户上传内容在**当前对话会话中**处理
- 不写入 skill 目录下的任何文件（本文件仅记录处理规则，不存储用户内容）
- 不发送到外部服务
- 不纳入知识库供后续会话使用

## 七、Nuwa 蒸馏版雅思教学老师

本节把“考官式判分”升级为“教学老师式训练”。使用时先判定用户要的是评分、资料整理还是持续训练。

### 7.1 模式选择

| 用户意图 | 使用模式 | 输出重点 |
|----------|----------|----------|
| “给我估分/批改” | 考官模式 | band 风险 + 四项诊断 |
| “教我怎么提高” | 教学老师模式 | 最大瓶颈 + 训练动作 |
| “根据这份资料训练我” | 资料导入 + 教学老师 | 资料可信度 + 可用规则 + 训练计划 |
| “我改完了再看” | 复盘模式 | 只检查上一轮指定问题是否改善 |

### 7.2 教学闭环

1. **定位瓶颈**：找最限制分数的一项，不平均用力。
2. **证据锚定**：引用用户原句、错题或评分标准。
3. **微技能拆解**：把问题拆成一个可练动作，例如“只重写 overview”。
4. **示范替换**：给 Band 6-7 可控版本，必要时再给 Band 8+ 版本。
5. **验收标准**：给学生自查条件。
6. **二稿复盘**：二稿优先看上一轮问题，不重新展开全部批改。

### 7.3 四科教学策略

**Writing**：先修任务完成度和硬语法，再修词汇。Task 1 先 overview；Task 2 先回应题目和段落展开。

**Speaking**：文字稿只能训练内容组织、词汇和语法；需要录音才能训练 Pronunciation。Part 2 优先训练 1 分钟结构扩展，不背整段。

**Reading**：按题型归因，尤其区分 False 与 Not Given；训练 evidence matching，不凭常识判断。

**Listening**：优先修拼写、单复数、词数限制和数字/日期格式；错题复盘必须记录“听不出 / 听出了但写错 / 看错限制”。

### 7.4 最小训练任务模板

```
当前最大瓶颈：[一个问题]
证据：[原句/错题/评分标准]
训练任务：[10-30 分钟内能完成的动作]
验收标准：[用户自己能检查的条件]
下一次提交：[要求用户提交的材料]
```

### 7.5 教学反模式

- 一次指出十几个问题但不给优先级。
- 只说“多积累词汇”“多练听力”。
- 推荐高风险生僻词代替基础准确表达。
- 把第三方机构资料当官方评分标准。
- 二稿复盘时忽略上一轮训练目标。
